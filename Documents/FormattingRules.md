# Formatting Rules

## Variable Format Rules
Our rule about coding is snakeCase with member variables being m_VariableName, and static variables being s_MemberName.

## Git Commit Rules
Our rule about `git commit` is using the format
    `* add: What has been added` or `* fix: Bug that is causing this or that`

## General C/C++ Format
Our rule about `C/C++` file format is linux's format that you can see form .clang-format and coupled with .clang-tidy.
